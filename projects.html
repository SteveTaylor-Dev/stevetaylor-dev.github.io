Older Projects

Growing up surrounded by computers and game consoles from a very early age, I developed a lifelong passion for technology and game design that's still strong today. Since having access to a computer I've looked for ways to create anything I can; art, quality-of-life tools (like batch scripts for file management), game modifications, and eventually my own game frameworks. A large driving factor in all of this was the first game I was introduced to; Sonic the Hedgehog. I'd probably usually try to distance myself from a series aimed at kids, but being my introduction to any sort of gaming experience, it's come to define what, and how, I think about video games. Being built around an incredibly physics driven design, the idea of building something using similar mechanics always felt too large of a task, but one that's been a goal of mine for as long as I can remember.

Clockwork Cascade (2010) - Multimedia Fusion

One of my earliest attempts at achieving this was using a GUI based programming tool, Multimedia Fusion, which had many prebuilt gameplay frameworks available. I started with one of these frameworks, while modifying the core behaviour and adding to it, to build a game using original art and assets, while keeping the fundamental aspects of Sonic gameplay. At the time, I was taking a more designer role than programmer, being focused on the visual and audio design, along with ensuring the game mechanics and level layout flow with each other intuitively. While this was a decent outlet for designing an experience I'd always wanted to make, unless I gained the proper programming skills I was still always going to be limited in scope.

Crumbling Canyon (2011) - iD Tech 1 (SRB2)

Until gaining these skills, I continued working towards making a game using prebuilt tools. One example, from 2011, was a series of maps designed for a Sonic inspired modification built with the Doom Legacy Engine (iD Tech 1). Again focusing on the level layout, sound, and visual design, this was my first attempt at working with 3D design tools (though in a very primitive form). Using trigger collision systems and building gameplay scenarios using pre-defined functions was a step closer towards understanding some basic programming principles (specifically for a 3D space) before finally attempting it myself properly.
